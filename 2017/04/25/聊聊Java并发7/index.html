发表于 2017-04-25 | 分类于 Java提升

并行设计模式属于设计优化的一步扥，它是对一些常用的多线程结构的总结和抽象。与串行程序相比，并行程序的结构通常更为负责。因此合理的使用并行模式在多线程开发中更具有意义，在这里主要介绍Future、Master-Worker和生产者-消费者模型。

Future模式

Future模式有点类似于商品订单。比如在网购时，当看重某一件商品时，就可以提交订单，当订单处理完成后，在家里等待送货上门即可。后者说我们更形象的我们的Ajax请求的时候，页面是异步的进行后台处理，用户无需一直等待请求的结果，可以继续浏览或者其他内容。

Future应用场景，比如内部操作非常耗时的，执行很多事情，总之很复杂，同步的方式执行很慢，那么就用Future模式，异步的执，而不是一个线程同步一步步执行，每个小业务都有自己的线程，利用空间换时间，并行执行！

Master-Worker模式

Master-Worker模式是常用的并行模式，他的核心思想是系统由两类金恒写作工作：master进程和Worker进程。Master负责接收和分配任务，Worker负责处理子任务。当哥哥Worker紫禁城处理完成后，会将结果返回给master，由master做归纳总结。其好处是能将一个大的任务分解成若干的小人物，并行执行，从未提高系统的吞吐量。

master-worker是并行计算的设计模式，master负责接收分配任务，worker复制处理任务，最后master做一个总结归纳，从而提高系统的吞吐量。Hadoop和Storm主从业务逻辑都是这样实现的。

应用场景:当我们数据量不是很大的时候就可以使用master-worker设计模式去解决文问题。

生产者-消费者模式

生产者-消费者模式也是一个非常经典的多线程模式，我们在实际开发应用非常广泛的思想理念。在生产者-消费者模式中：通常有两类线程，即若干个生产者的线程和若干个消费者线程。生产者线程负责提交用户请求，消费者线程则负责具体处理生产者提交的任务，在生产者和消费者之间通过共享内存缓存区进行通信。
